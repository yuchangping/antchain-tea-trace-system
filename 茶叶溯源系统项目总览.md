## 2. 这个项目到底在做什么

一句话概括：

**这个项目要做的是一个“基于区块链的茶叶全流程溯源系统”，让茶叶从采摘到销售的全过程可以被录入、查询、展示，并且把关键摘要信息写到区块链上，提升溯源信息的可信度。**

它不是单纯做一个管理后台，也不是单纯做一个区块链实验，而是想把下面几件事串成一条完整链路：

1. 后台人员录入茶叶批次和各环节信息
2. 系统把详细业务数据保存到 MySQL
3. 系统把关键摘要数据写到区块链
4. 系统自动生成二维码
5. 消费者扫码后能看到茶叶的完整溯源过程
6. 演示时还能展示交易哈希、区块号、链上凭证

所以，这个项目的核心不是“上链”本身，而是**做出一个可以完整演示的茶叶溯源闭环系统**。

---

## 3. 这个项目要解决什么问题

茶叶这类农产品在实际流通过程中，常见问题是：

- 来源信息不透明
- 生产、加工、检测、物流、销售信息分散
- 消费者很难判断产品是否真实可信
- 即使有记录，也不容易证明记录有没有被篡改

这个项目的解决思路是：

- **链下存详情**
  - 用 MySQL 保存批次、环节记录、文件、账号等完整业务数据
- **链上存摘要**
  - 用区块链保存批次摘要、环节摘要、操作人、时间等关键凭证
- **扫码做展示**
  - 用二维码把后台录入和消费者查询串起来

最终效果是：后台能录、系统能存、链上能证、前端能看、扫码能查。

## 4. 项目的目标系统长什么样

整个系统分为四层：

### 4.1 前端展示层

- 管理后台
- 消费者扫码查询页

管理后台负责录入和管理，扫码页负责展示和查询。

### 4.2 后端服务层

- 提供 REST API
- 处理登录鉴权
- 处理业务数据
- 处理文件上传
- 生成二维码
- 调用区块链服务

### 4.3 区块链层

- 本地链环境使用 Ganache
- 智能合约使用 Solidity + Hardhat
- 负责保存批次摘要和环节摘要

### 4.4 数据库与文件存储层

- MySQL 保存详细业务数据
- 本地目录保存图片、报告、二维码等文件

项目的核心设计原则是：

**详细信息不上链，关键摘要上链。**

## 5. 这个项目要实现哪些核心功能

按照业务闭环来看，核心功能可以分成 8 类。

### 5.1 用户登录与身份进入

- 用户登录后台
- 获取当前用户信息
- 基于角色进入系统

### 5.2 茶叶批次管理

- 创建茶叶批次
- 查询批次列表
- 查看批次详情

批次是整个项目的主线，后面的所有环节记录都围绕 `batch_no` 展开。

### 5.3 溯源环节记录管理

项目固定围绕 6 个业务环节做记录：

- `harvest`：采摘
- `process`：加工
- `inspection`：检测
- `package`：包装入库
- `logistics`：物流运输
- `sale`：销售上架

每一条记录都要支持：

- 标题
- 内容摘要
- 扩展 JSON 详情
- 操作人
- 操作时间
- 附件地址
- 数据摘要哈希
- 上链状态

### 5.4 文件上传

- 上传图片
- 上传检测报告
- 保存文件元数据
- 在批次或记录详情中展示附件

### 5.5 二维码生成

- 创建批次后生成二维码
- 二维码跳转到公开查询页
- 扫码后直接按批次号查询

### 5.6 区块链上链与验证

- 创建批次时生成批次摘要并上链
- 新增环节记录时生成记录摘要并上链
- 保存交易哈希、区块号、链状态
- 查询链上批次信息
- 验证链上记录是否一致

### 5.7 后台展示

- 首页仪表盘
- 批次列表页
- 批次详情页
- 新增记录页
- 链上记录页

### 5.8 消费者扫码查询

- 无需登录
- 展示批次基本信息
- 展示完整时间轴
- 展示链上交易信息
- 适合手机端浏览和课堂演示

## 7. 这个项目的核心业务流程

从使用流程看，项目最终应该跑通下面这条主链路：

1. 管理员或业务人员登录后台
2. 创建一个新的茶叶批次
3. 系统保存批次信息到数据库
4. 系统生成批次摘要并写入区块链
5. 系统生成该批次的二维码
6. 后台继续录入采摘、加工、检测、包装、物流、销售记录
7. 每新增一条记录，系统都保存数据库详情并写入链上摘要
8. 消费者扫码进入公开查询页
9. 页面展示该批次的完整时间轴、附件和链上凭证
10. 演示时展示交易哈希、区块号和上链状态

这就是项目真正想完成的“全流程溯源闭环”。

## 10. 可以把这个项目理解成什么

如果要用最容易讲给别人听的话来解释，可以这样说：

**这是一个“茶叶版的全流程履历系统”。**

它像是在给每一批茶叶建立一份可以被追踪、被展示、被验证的数字档案：

- 数据库负责保存完整履历
- 区块链负责保存关键凭证
- 二维码负责把管理端和消费者端连接起来

所以它本质上是：

- 一个溯源项目
- 一个前后端分离项目
- 一个区块链存证项目
- 一个适合作为课程展示的综合实战项目
